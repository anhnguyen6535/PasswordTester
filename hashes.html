What does this mean?

“MD5 hash” is initially designed as a cryptographic hash function.
The system convert user's password to a line of MD5 hash numbers, and store them into a root file.
Usually, we cannot convert MD5 hash back to the corresponding original password.
However, the MD5 hashes of many common passwords have already been posted online.
If you used a common password, someone would have a higher chance to convert the hash back to the original password using online data.
Thus, using common passwords (even with some "booster" modifications) is still vulnerable to cyberattacks.
